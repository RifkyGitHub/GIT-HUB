note :
(.) titik = keseluruhan / semua file
nama_file = file tertentu

vocabulary :
-f = force
-d = delete
-m = move / rename
-c = create
-u = upstream
-v = verbose (detail)
-a = all
-r = remote
diff = difference
init = initialization
log = riwayat git
staged = area siap commit
hash = kode unik commit
HEAD = posisi commit / branch aktif sekarang
origin = nama default remote github

========================
DASAR GIT
========================

git init : membuat repository git baru
git status : melihat status file / branch saat ini
git status -s : status singkat
git add . : memasukkan semua file ke staging area
git add nama_file : memasukkan file tertentu ke staging area
git add -A : stage semua perubahan
git commit -m "pesan" : menyimpan perubahan ke riwayat commit
git commit -am "pesan" : add + commit file yang sudah tracked
git config --global --edit : edit konfigurasi detail
git config --global user.name "nama" : set username
git config --global user.email "email" : set email

========================
PERUBAHAN FILE
========================

git diff : perubahan belum di-stage
git diff --staged : perubahan sudah di-stage
git diff HEAD : perubahan dibanding commit terakhir
git clean -f : hapus file untracked
git clean -fd : hapus file untracked + folder
git restore . : kembalikan semua perubahan
git restore nama_file : kembalikan file tertentu
git restore --staged nama_file : unstage file
git checkout -- nama_file : cara lama restore file

========================
RIWAYAT COMMIT
========================

git log : seluruh riwayat commit
git log --oneline : versi singkat
git log --graph : tampilan cabang
git log --oneline --graph --all : riwayat lengkap graph
git show code_hash : detail commit tertentu
git show HEAD : lihat commit terakhir
git diff hash1 hash2 : bandingkan 2 commit
git blame nama_file : lihat siapa ubah tiap baris

========================
BRANCHING
========================

git branch : melihat daftar branch
git branch namabranch : membuat branch baru
git branch -a : lihat branch lokal + remote
git branch -r : lihat branch remote
git switch namabranch : pindah branch
git switch -c namabranch : buat branch + pindah
git checkout namabranch : pindah branch / commit
git checkout -b namabranch : buat branch + pindah
git branch -m namabaru : rename branch aktif
git branch -d namabranch : hapus branch aman
git branch -D namabranch : hapus branch paksa
git merge namabranch : gabungkan branch ke branch aktif
git merge --abort : batalkan merge conflict
git rebase main : pindahkan branch di atas main

========================
REMOTE / GITHUB
========================

git remote -v : lihat remote
git remote add origin link_repo : hubungkan github
git remote remove origin : hapus remote
git push -u origin main : push pertama + set upstream
git push : push commit terbaru
git push origin namabranch : push branch tertentu
git pull : ambil update + merge
git fetch : ambil update tanpa merge
git clone link_repo : copy repo dari github

========================
RESET / UNDO
========================

git reset --soft hash : mundur commit, staged tetap ada
git reset --mixed hash : mundur commit ke working dir
git reset --hard hash : mundur commit, perubahan hilang
git revert hash : membatalkan commit dengan commit baru
git rm nama_file : hapus file + stage
git mv lama baru : rename / pindah file

========================
STASH
========================

git stash : simpan perubahan sementara
git stash list : lihat daftar stash
git stash apply : pakai stash terakhir
git stash pop : pakai stash + hapus dari list
git stash drop : hapus stash

========================
TAG
========================

git tag : lihat tag
git tag v1.0 : buat tag
git push origin v1.0 : push tag

========================
INFO PENTING
========================

HEAD : posisi sekarang
main : branch utama
origin : nama remote default
tracked file : file yang sudah dikenal git
untracked file : file baru belum di-add
modified : file diubah
staged : siap commit
merged : sudah digabung
conflict : bentrok perubahan saat merge
git cherry-pick (hash) : Cherry-pick dipakai kalau lu mau pilih commit spesifik, bukan seluruh branch.
